Welcome

A Markovian Dependability Model with Cascading Failures

Mihir Sanghavi and Sashank Tadepalli, undergraduate students at New Jersey Institute of Technology alongside Professor Marvin Nakayama, have developed a cutting-edge implementation of DECAF.  Shown below are the results of various tests run across various versions of this software.  We provide Hardware and Software specifications so these tests can be recreated at a future date.  All time-based results are averaged over 3 trial runs (except for 1944+ State Models).

Versions

Version 1.0 and 1.5 are the initial versions of DECAF. The essential algorithms and program logic is created in Version 1.0, as well as some naive implementations of the Tree Generation algorithm. Version 1.5 adds some performance enhancements in the form of caching. The caching occurs at an analytical level as well as software/programming level. Essentially, every transition from F to T is treated as a LikeTransition. When we accumulate all of the LikeTransitions, it becomes apparent that the program does not need to recreate trees for the same LikeTransition.

Version 2.0 takes it a step further in performance enhancements. A revolutionary new Tree Generation algorithm is put to the test, and the results are remarkable. Furthermore, smaller enhancements were made to State Generation (enumeration over component redundancies) as well as XML parsing (no more DOM).
